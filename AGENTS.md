# AGENTS.md v0.0.2

このファイルはAIエージェント向けの開発ガイドラインです。

## バージョン管理ルール

### バージョン表記について

- このファイルのタイトルにはバージョン番号（`v0.0.X`）を表記します
- ブランチ内で修正があるたびにパッチバージョン（3桁目）を更新してください
- 例: `v0.0.1` → `v0.0.2` → `v0.0.3`

### バージョン更新のタイミング

1. **パッチバージョン更新**: ブランチ内での修正・変更時
2. **マイナーバージョン更新**: 新機能追加時
3. **メジャーバージョン更新**: 破壊的変更時

## プロジェクト概要

FigmaのVariablesとNotionデータベースを同期するプラグインです。

### 技術スタック

- **言語**: TypeScript 5.9
- **UI**: React 19
- **ビルド**: Vite 7
- **プラットフォーム**: Figma Plugin API

### ディレクトリ構造

```
src/
├── plugin/           # Figmaプラグインのコア機能
│   ├── controller.ts # メインコントローラー
│   ├── handlers/     # 同期処理ハンドラー
│   └── utils/        # Variable操作ユーティリティ
├── ui/               # React UI
│   ├── components/   # UIコンポーネント
│   ├── services/     # Notion APIサービス
│   └── styles/       # スタイルシート
└── shared/           # 共通型定義・ユーティリティ
```

## 開発ガイドライン

### コーディング規約

1. **型安全性**: `any`型の使用は避け、適切な型定義を行う
2. **命名規則**: 
   - 変数・関数: camelCase
   - 型・インターフェース: PascalCase
   - 定数: UPPER_SNAKE_CASE
3. **ファイル命名**: 
   - コンポーネント: PascalCase.tsx
   - ユーティリティ: camelCase.ts

### セキュリティ

- APIキー・トークンは難読化して保存
- ログ出力では機密情報をマスキング
- Cloudflare Workers経由で通信

### テスト・ビルド

```bash
# 開発モード
npm run dev

# ビルド
npm run build

# リント
npm run lint

# 型チェック
npm run type-check

# テスト実行
npm run test

# テスト（ウォッチモード）
npm run test:watch
```
